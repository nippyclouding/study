# 외부 API 연동과 공식 문서 읽는 법

외부 API(다음 지도, 카카오 지도, 카카오 도서, 토스 결제 등)를 프로젝트에 연동할 때, 구체적인 연동 코드를 외우거나 단순히 블로그의 예제 코드를 복사해서 붙여넣는 것은 좋은 방법이 아닙니다. API마다 요구하는 인증 방식, 요청 규격, 응답 형태가 모두 다르며, 수시로 버전이 업데이트되기 때문입니다.

따라서 API를 연동하기 위해 가장 먼저, 그리고 가장 중요하게 해야 할 일은 **해당 API의 '공식 문서(Official Documentation)'를 읽는 것**입니다.

---

## 1. 왜 공식 문서를 읽어야 하는가?

*   **정확성과 최신성:** API 제공자가 직접 작성한 문서이므로 가장 정확합니다. 블로그 글은 작성 시점의 과거 버전을 기준으로 쓰여 있어 현재는 작동하지 않거나(Deprecated) 스펙이 변경되었을 수 있습니다.
*   **문제 해결의 단초:** 연동 중 에러가 발생했을 때, 에러 코드의 의미와 해결 방법, 호출 제한(Rate Limit) 등의 규정이 모두 공식 문서에 명시되어 있습니다.
*   **응용력 향상:** 문서를 읽고 해석하는 능력을 기르면, 처음 보는 어떤 외부 API(해외 결제, 날씨, 소셜 로그인 등)라도 스스로 연동할 수 있는 힘이 생깁니다.

---

## 2. 공식 문서 읽는 방법론 (핵심 체크리스트)

방대한 공식 문서를 처음부터 끝까지 소설 읽듯 읽을 필요는 없습니다. 아래의 순서와 핵심 요소들을 중심으로 문서를 파악하세요.

### ① 시작하기 (Getting Started / Quick Start) 확인
대부분의 공식 문서는 '시작하기' 가이드를 제공합니다. 여기서는 전체적인 연동 흐름을 가장 빠르게 파악할 수 있습니다.
*   **애플리케이션 등록:** API를 사용하기 위해 개발자 센터에 내 프로젝트를 등록하는 방법.
*   **API Key 발급:** 호출 권한을 증명하기 위한 고유 키(Key)를 발급받는 위치와 방법.

### ② 인증 및 권한 (Authentication & Authorization)
API를 호출할 때 내가 누구인지(혹은 정상적인 권한을 가진 사용자인지) 증명하는 방법입니다.
*   HTTP Header에 API Key를 넣어야 하는지? (예: `Authorization: Bearer {token}`)
*   OAuth 2.0 방식을 사용하여 토큰을 발급받고 갱신해야 하는지?

### ③ API 레퍼런스(Reference) 분석 - 요청(Request) 규격
원하는 기능을 사용하기 위해 제공자에게 **어떤 형식으로 데이터를 보내야 하는지** 파악합니다.
*   **Base URL 및 Endpoint:** 요청을 보낼 목적지 주소 (예: `https://dapi.kakao.com/v3/search/book`)
*   **HTTP Method:** 데이터를 조회(GET), 생성(POST), 수정(PUT/PATCH), 삭제(DELETE)할지 확인.
*   **파라미터 (Parameters):**
    *   **필수(Required) vs 선택(Optional):** 반드시 보내야 하는 값과 안 보내도 되는 값을 구분합니다.
    *   **위치:** Query String으로 보낼지(`?query=hello`), Request Body(JSON)에 담아 보낼지, Path Variable(`/users/123`)로 보낼지 확인합니다.

### ④ API 레퍼런스(Reference) 분석 - 응답(Response) 규격
요청이 성공했을 때 **어떤 형태의 데이터가 돌아오는지** 파악합니다.
*   **데이터 포맷:** 주로 JSON 형태를 띠며, 내가 필요한 데이터(예: 도서명, 썸네일 이미지 URL)가 JSON 구조의 어느 위치(Depth)에 있는지 미리 확인합니다.
*   **성공/실패 여부:** HTTP 상태 코드(예: 200 OK, 400 Bad Request) 및 응답 본문의 상태 필드를 확인합니다.

### ⑤ 에러 코드 (Error Codes)
연동 과정에서 마주치는 수많은 에러들을 해결하는 열쇠입니다.
*   401(인증 실패), 403(권한 없음), 404(잘못된 엔드포인트), 429(호출 한도 초과) 등 API 제공자가 정의한 에러 코드표를 확인하여 원인을 특정합니다.

### ⑥ 샌드박스(Sandbox) 및 예제 코드 활용
*   문서에서 제공하는 **테스트 도구(API Test / Sandbox)** 나 **cURL 예제**를 활용해 코드를 작성하기 전, 요청과 응답이 정상적으로 이루어지는지 먼저 테스트합니다.
*   Postman 같은 API 테스트 툴을 활용하여 미리 호출해 보면 구조를 이해하는 데 큰 도움이 됩니다.

---

## 요약

외부 API 연동은 코드를 암기하는 것이 아니라, **제공자가 정해둔 규칙(공식 문서)을 이해하고 그 규칙에 맞게 나의 요청을 조립(퍼즐 맞추기)하는 과정**입니다. 처음에는 낯설고 어렵게 느껴질 수 있지만, 위에서 언급한 구조(인증 -> 요청 스펙 -> 응답 스펙)에 익숙해지면 어떤 문서를 보더라도 빠르게 필요한 정보를 찾아 연동할 수 있습니다.
